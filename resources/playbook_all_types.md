# 骨架一：产品犀利评测

## 结构模板
```
【我用了[时间]，我的真实感受是：】
直接说：好用 / 难用 / 值得买 / 不值得
不绕弯子，第一段就给结论

【它做对了什么（哪怕我整体不推荐）】
公平评测原则：找出产品的真实亮点
- 用具体的使用场景说明（「在[场景]下，它[表现]」）

【它做错了什么（即使整体推荐）】
更重要的部分：真实的缺点
- 不是「可以更好」这种废话
- 而是「它在[场景]下让我[具体受挫]，因为[具体原因]」

【和竞品的真实对比】
不是参数对比表，而是「在同样的[使用场景]下，[竞品]的体验是这样，而[本品]的体验是这样」

【买不买的建议（分人群）】
「如果你是[人群A]：买」
「如果你是[人群B]：不买」
清晰、直接，不装中立
```

# 骨架二：行业批判观察

## 结构模板
```
【我要说一件[行业]里没人说的事】
品玩最典型的开场：
「[行业/公司]一直在说[X]。但我们长期观察下来，真相是[Y]。」

【炒作的地方在哪里】
具体指出：哪些数据在被刻意夸大？
哪些「进展」是PR稿而不是实质突破？

【没被炒作到的真实情况】
相对客观地呈现行业现状：
- 真实的市场规模（vs. 官方夸大的数字）
- 真实的用户行为（vs. 企业描述的用户行为）

【但这件事也不是一无是处】
品玩的「有品」体现在这里：批判不是为了批判
在批评之后，说清楚「这件事真正有价值的地方在哪里」

【我的判断】
用第一人称给出清晰结论：
「我们认为，[这件事/这个行业]的真实状态是[判断]。
不是[乐观派说的]，也不是[悲观派说的]，而是[品玩的判断]。」
```

# 骨架三：科技生活方式

## 结构模板
```
【从一个日常场景开始】
「上周，我发现自己[日常使用某技术/产品的具体场景]。然后我开始想……」

【这件科技小事背后更大的变化】  
从个人体验拔高到趋势洞察：
「这不只是[A]，这是[更大趋势]的一个缩影。」

【更多人也有一样的感受】
几个真实的社交媒体/动态引用，证明这是共同体验

【科技正在改变什么，以及没在改变什么】
有品玩的辩证视角：
技术有时被高估（什么没变），也有时被低估（什么悄悄变了）

【你可以现在就试试的建议】
落到读者：「如果你也[有同样情况]，可以试试[具体建议]。」
```

# 骨架四：全球科技对比

## 结构模板
```
【海外已经发生了这件事】
用硅谷/欧洲/日本的一个真实现象开场，作为「镜子」

【中国的对照情况】
「在中国，这件事的现状是……」
横向比较，找出差距和差异，但不是简单的「中国落后」

【为什么存在这个差异】
结构性分析：技术？政策？市场结构？消费者习惯？

【中国市场的独特性（不是矮化）】
品玩的全球视角不是「美国是标准」：
「但中国市场也有一些[海外市场没有]的条件/优势」

【对读者的启示】
这个跨国比较给中国消费者/从业者带来什么值得借鉴的地方
```

# 骨架五：趋势犀利观察

## 结构模板
```
【这件事，大家说的和真实的有多大距离】
品玩观察趋势的角度：先测温，主流判断和真实情况的差距有多大

【真实信号 vs. 噪声信号的识别】
区分：哪些是有效信号（数据支持，可被验证）
哪些是噪声信号（PR / 热度 / 资本炒作）

【早于主流的观察】
品玩的价值定位：比主流媒体更早指出某个趋势的转折
「我们3个月前就注意到[这个信号]，现在它开始[印证]了」

【趋势的真实受益者和受损者】
不是泛泛的「影响所有人」，而是：
「如果这个趋势成立，[A类人]会得到[X]，而[B类人]将面临[Y]」

【行动建议：你现在应该关注的3件事】
具体、可追踪的建议，而不是「持续关注这一领域」
```
